To deploy the system and run a meaningful trial, we need the following before the setup call:

**Access**
- A2000 API credentials (preferred) or A2000 admin login for screen automation fallback

**Sample Data**
- 3–5 representative PO emails with attachments (from Walgreens or other regular buyers)
- These can be real historical POs — we use them to configure the parser before going live

**Your Time**
- 30-minute setup call to walk through one order manually and confirm the integration approach

- Two 20-minute check-in calls during the trial (we propose the times)

**Hardware**
- A Windows computer that can run during business hours (the computer running A2000 works fine)
- No hardware purchase required for the trial. If you ever want to run the AI fully offline on dedicated hardware, that's an upgrade we can do later.

## Trial Success Criteria

At the end of 30 days, we evaluate the trial against these benchmarks:

| Criteria | Target |

| PO processing accuracy | End-to-end, correct order entry with no manual correction needed |

## Data Handling During the Trial

- All business data is stored locally in an encrypted vault on your machine
- The AI providers Hermes uses for parsing (Anthropic / OpenAI) are bound by SOC 2 Type II certifications and Data Processing Agreements that prohibit storage, training, or sharing of your data
- Encrypted backups are taken every 6 hours to a separate provider so a single failure cannot cause data loss
- OASIS AI may access your system remotely for setup and troubleshooting — only with your explicit permission per session
- We do not retain copies of your POs, order data, or customer information
- At trial end (pass or fail), no data is transferred out — it remains in your encrypted local database, which is yours to keep
