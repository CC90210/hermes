# memory/ — Per-Deployment State

> This directory structure is committed. The runtime contents are not (gitignored per deployment).

## Committed Files (Product Repo)

These files are committed to `CC90210/hermes` and ship with every deployment:

| File | Mutability | Purpose |
|------|-----------|---------|
| README.md | GOVERNED | This file |
| MEMORY_INDEX.md | FREELY MUTABLE | Catalog of all memory files |
| LONG_TERM.md | GOVERNED | High-confidence persistent facts |
| PATTERNS.md | GOVERNED | Probationary and validated patterns |
| MISTAKES.md | GOVERNED | Root causes and prevention rules |
| DECISIONS.md | FREELY MUTABLE | Business rules Emmanuel has locked in |
| CUSTOMERS.md | FREELY MUTABLE | Customer-specific quirks and compliance |
| ACTIVE_TASKS.md | EPHEMERAL | Current work in flight |
| SESSION_LOG.md | EPHEMERAL | Per-session activity log |
| SELF_REFLECTIONS.md | EPHEMERAL | Structured failure analysis |
| PROPOSED_CHANGES.md | EPHEMERAL | Queue for semi-mutable file proposals |

## Runtime Files (Per Deployment — NOT Committed)

Generated by the running pipeline. Gitignored.

- `failure_patterns.jsonl` — structured records of parsing failures (for Ollama prompt tuning)
- `audit_exports/` — CSV/JSON exports of audit_log for operator review
- `pattern_detections.json` — aggregated failure pattern counts


## Notes for OASIS AI

During maintenance visits:
- Query `failure_patterns.jsonl` for recurring parse errors → tune Ollama prompts
- Check `audit_exports/` for any operator-requested exports
- Review MISTAKES.md and PATTERNS.md for growth opportunities
- Never commit `*.db` files back to the product repo (real customer data)
