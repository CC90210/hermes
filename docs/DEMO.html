The loop Hermes closes

Five steps. Zero touch. Under a second.

When a PO arrives, Hermes runs the whole workflow himself. You see a summary in the morning, not a to-do list.

STEP 01
📧 PO Received
Hermes watches your Outlook inbox. Detects incoming POs by subject, attachment type, or body content. Grabs the PDF, Excel, or EDI X12 file.
STEP 02
🤖 PO Parsed
A local AI model reads the document and extracts the PO number, customer, line items, prices, and ship dates. All computation stays on your computer.
STEP 03
📦 Order Entered
Hermes enters the order into A2000 via EDI 850 or the vendor API. He also sends an EDI 855 PO acknowledgment back to the buyer within the 24-hour window.
STEP 04
🏷️ Labels + ASN
GS1-128 carton labels print to your Zebra thermal printer. The EDI 856 ASN transmits four hours before the DC cutoff. Walgreens never fines you for a missed window.
STEP 05
🧾 Invoice Out
The invoice generates from A2000, gets attached to an email, and goes back to the buyer. Every action is logged. You never touched a keyboard.
When everything is clean, you see one line in your morning briefing: "7 POs processed overnight, $48,220 total, all clean."

The architecture

What's running under the hood.

Hermes has two modes running side by side, sharing the same brain (SQLite database + audit log + memory files):

Your Outlook Inbox │ ▼ Email Agent (poll every 5 min — IMAP/Exchange) │ ▼ PO Parser (local Ollama LLM — on your machine) │ ▼ Manager Bot (orchestrate · retry · escalate · audit) │ ┌────────────────┬───────────┼────────────────┐ ▼ ▼ ▼ ▼ POS Adapter EDI 855 Ack GS1-128 Labels EDI 856 ASN A2000 (4 modes) 24-48 hr Zebra thermal Pre-ship TX │ ▼ Invoice Agent (SMTP back to customer) │ ▼ EDI 820 Parser (remittance + chargeback detection) │ ▼ Chargeback Tracker (4-week dispute window alert)

Background Hermes

The Python pipeline. Runs every 5 minutes on Windows Task Scheduler. Watches your inbox, enters orders, transmits EDI, sends invoices. You don't interact with this one. It just works.

IDE Hermes

The one you talk to. Opens in Claude Code on your desktop. Answers questions, drafts emails, runs reports, re-runs failed orders. Shares the same database as the pipeline, so you see live state.

  • Acknowledges a PO with changes (he escalates — you decide)

  • Uninstalls or modifies system applications
  • Shares or transmits data outside your local network
